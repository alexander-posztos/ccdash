# ccdash

A terminal dashboard for **all** your Claude Code projects at once - each with a
recap, git state, and Enter to resume.

It's easy to lose track when you're working across a lot of projects with Claude
Code. ccdash gives you a single overview of your recent ones: each project's
sessions, its git state, and a quick recap, generated by Claude Haiku, of where
you left off. The recap is built from your last session transcript, the git
state, and any recent `STATUS.md` / `HANDOFF.md` / `TODO.md` (and similar) notes
in the repo. The transcript leads, so a stale note will not throw the recap off.
It is cached on disk so the dashboard opens instantly and refreshes in the
background as projects change.

Claude Code's built-in `/resume` only works inside the project you're already in.
ccdash spans all of them at once, so you can open it from any directory and still
see every project, then jump straight back in. Press Enter to resume a session,
or `n` to start a fresh one.

It is read-only over `~/.claude`, never writes to your repos, and `--offline`
derives recaps locally with no network call.

Hide projects with `x`; the `a` scope cycles `active -> all -> hidden`. Projects
whose directory is gone are flagged as missing and come back once the directory
returns.

## Install

```sh
go install github.com/alexander-posztos/ccdash@latest
```

Or download a prebuilt binary from the
[releases page](https://github.com/alexander-posztos/ccdash/releases), or build
from source:

```sh
git clone https://github.com/alexander-posztos/ccdash
cd ccdash
CGO_ENABLED=0 go build -o ccdash .
```

**Requirements:** macOS or Linux, `git`, and the
[`claude`](https://docs.claude.com/en/docs/claude-code) CLI (used to resume
sessions and to generate live recaps). Building from source or via `go install`
needs Go 1.25+.

## Keys

```
j / k  ↑ / ↓    move
tab / → ←       switch pane / drill into sessions
g / G           jump to first / last session
pgup / pgdn     scroll the detail pane
enter           drill in · resume session
l               resume the latest session
n               new session in project
r               regen the selected recap
x               hide / unhide project
a               scope: active (30d) / all / hidden
/               filter by name
? esc q         help · back · quit
```

## CLI

```
ccdash                        interactive TUI (default)
ccdash --offline              TUI, but derive recaps locally (never call claude)
ccdash --singleton            if ccdash is already open, exit 0 (let the WM focus it)
ccdash -l, --list             plain-text dump of projects + recaps, then exit
ccdash --list --all           include inactive projects too
ccdash --json                 machine-readable project index as JSON
ccdash hide <dir>             hide a project so it drops out of the dashboard
ccdash unhide <dir>           unhide a previously hidden project
ccdash version                print version, commit, and build date
```

## Launch from anywhere

You can run `ccdash` from anywhere - there's no need to `cd` into a project
first. The easiest way to use it is to map `ccdash` to a global hotkey. I bind it
to a hyper key (Caps Lock remapped to Ctrl+Alt+Cmd+Shift) so `hyper + g` opens
ccdash from any app or directory, and I jump straight to the project or session I
want.

## Themes

ccdash renders over your terminal's own background and ships three dark palettes,
chosen with `CCDASH_THEME`:

- `slate` (default) - cool, neutral blue-gray
- `ember` - warm amber
- `tokyonight` - the Tokyo Night "night" variant

Built-in names are case- and separator-insensitive (`tokyonight`, `tokyo-night`,
and `Tokyo Night` all match).

### Bring your own

`CCDASH_THEME` also accepts a JSON theme file: a path
(`CCDASH_THEME=~/themes/gruvbox.json`) or a bare name under
`~/.config/ccdash/themes/` (override with `CCDASH_THEME_DIR`). It is seven
`#RRGGBB` color slots.

## Configuration

| Variable | Default | Meaning |
| --- | --- | --- |
| `CCDASH_ACTIVE_DAYS` | `30` | a project counts as "active" if used within this many days |
| `CCDASH_MODEL` | `haiku` | model passed to `claude -p` for recaps |
| `CCDASH_THEME` | `slate` | palette name or JSON theme file (see Themes) |
| `CCDASH_OFFLINE` | unset | when set (`1`/`true`), derive recaps locally; never call `claude` |
| `CCDASH_CLAUDE_BIN` | auto | explicit path to the `claude` binary |
| `CCDASH_CLAUDE_DIR` | `~/.claude` | Claude Code data directory |
| `CCDASH_CACHE_DIR` | OS cache dir | where recaps are cached |
| `CCDASH_PREFS_FILE` | OS config dir (`ccdash/prefs.json`) | hidden-project prefs; in the config dir so a cache wipe never loses them |
| `CCDASH_THEME_DIR` | `~/.config/ccdash/themes` | directory searched for bare-name JSON theme files |
| `CCDASH_RECAP_TIMEOUT` | `90` | seconds to wait for a `claude -p` recap before timing out |

## Layout

- `main.go` - entrypoint and build-info wiring.
- `cmd/` - the CLI surface (cobra): root + plain modes, `version`, `hide`/`unhide`.
- `internal/config` - environment knobs and path resolution.
- `internal/engine` - read-only data + recap layer over `~/.claude` and git (no TUI deps).
- `internal/prefs` - writable hidden-project prefs store (`CCDASH_PREFS_FILE`).
- `internal/tui` - the Bubble Tea / Lipgloss dashboard and theming.

## Status

ccdash is a single static Go binary (Bubble Tea / Lipgloss). macOS and Linux are
supported. On Windows the read-only dashboard works, but resuming and starting
sessions (which re-exec a shell) are not supported. All paths are overridable via
the `CCDASH_*` variables above.
